## 웹 개발 방식
웹 개발 방식에는 정적 컨텐츠, MVC와 템플릿 엔진, API가 있다. 이 중 정적 컨텐츠 개발 방식에 대해 먼저 알아보자.

## 정적 컨텐츠
정적 컨텐츠 방식은 서버에서 작동하는 것 없이 파일을 그대로 웹 브라우저에 띄우는 것이다. 스프링 부트는 resources/static 폴더에 html 파일을 넣어놓으면 정적 페이지로 사용할 수 있다.

![image](https://user-images.githubusercontent.com/60471550/182162665-18f7525c-b2fa-496b-a912-a319a14396d5.png)

```html
<!DOCTYPE HTML>
<html>
<head>
    <title>static content</title>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
</head>
<body>
정적 컨텐츠 입니다.
</body>
</html>
```
### <http://localhost:8080/hello-static.html>
![image](https://user-images.githubusercontent.com/60471550/182163380-3bc4d162-be6f-45a6-bfe3-83b863af8ed6.png)

## 동작 방식
![image](https://user-images.githubusercontent.com/60471550/182163728-21469c4d-0ff6-4fa9-86b8-391ba884cffe.png)

웹 브라우저에서 localhost:8080/hello-static.html을 호출하면 톰캣 내장 서버에서 요청을 받아 스프링에 넘긴다. 스프링은 먼저 매핑된 컨트롤러가 있는지 확인하고 없을 경우 resources/static을 확인하여 hello-static.html 있으면 웹 브라우저에 반환해준다.
